A self-hosted deployment runs the studio, the API, the agent runner, and their datastores on your
own infrastructure. It has every feature of Agenta Cloud except the governance features listed
under [Editions](#editions).

## Start here

- [Quick start](/self-host/quick-start) runs the full stack locally with Docker Compose.
- [Deploy on a remote server](/self-host/guides/deploy-remotely) puts that stack on a server you own.
- [Deploy to Kubernetes](/self-host/guides/deploy-to-kubernetes) runs it from the Helm chart.
- [Deploy on Railway](/self-host/guides/deploy-on-railway) is a community-maintained deployment. There is
  no official support for it, and fixes depend on contributor availability.

## Where agents run

Agents execute in the runner service, in one of two sandboxes. Pick one before you open the
deployment to other people:

- [Run agents in a cloud sandbox (Daytona)](/self-host/agent-execution/daytona) gives every run its
  own isolated cloud sandbox. Use it for multi-user deployments.
- [Run agents locally](/self-host/agent-execution/run-agents-locally) executes agents inside the
  runner container. Local runs are the default, and they are not isolated from each other, so use
  them on a deployment you run for yourself or for a trusted team. They are also the only runs that
  can use [your own Claude, Pi, or ChatGPT subscription](/self-host/use-your-own-subscription)
  instead of a model API key.

[Sandbox isolation and security](/self-host/agent-execution/sandbox-isolation-and-security) covers
the difference in full. [Customize the agent runtime](/self-host/agent-execution/customize-the-agent-runtime)
adds tools, folders, or compute to either one.

## Editions {#editions}

- **OSS** is the open-source edition, under the
  [MIT license](https://github.com/Agenta-AI/agenta/blob/main/LICENSE). It covers the studio, the
  API, the runner, and both sandbox providers. Every page in this section applies to it.
- **EE** is the enterprise edition. It adds the governance features: SSO, roles, and
  [access controls](/self-host/guides/configure-sso). It is licensed separately.
