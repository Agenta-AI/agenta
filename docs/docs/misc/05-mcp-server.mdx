The Agenta MCP server lets MCP-capable agents author resources in an Agenta project through the Agenta API. It exposes a curated v1 surface over the confirmed `/simple/*` endpoints and intentionally does not start evaluation runs, deploy applications, manage billing or organizations, expose traces, or provide public multi-tenant OAuth.

All requests use one project-scoped Agenta API key configured on the MCP server. Treat hosted HTTP deployments as internal-only unless you add a separate per-user auth layer.

## Install from source

```bash
git clone https://github.com/Agenta-AI/agenta.git
cd agenta/clients/mcp-python
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -e .
```

## Configure Agenta

Set the API key and API base URL for the project you want the MCP server to author in:

```bash
export AGENTA_API_KEY="agenta-api-key"
export AGENTA_API_URL="https://cloud.agenta.ai/api"
export AGENTA_AUTH_SCHEME="ApiKey"
```

For EU Cloud, use:

```bash
export AGENTA_API_URL="https://eu.cloud.agenta.ai/api"
```

For self-hosted Docker Compose sibling deployment, use:

```bash
export AGENTA_API_URL="http://api:8000"
```

The package includes `.env.example`, but the server does not auto-load `.env` files. If you use one, source it before starting:

```bash
cp .env.example .env
$EDITOR .env
set -a && source .env && set +a
```

Verify the API connection with the read-only smoke test:

```bash
python3 scripts/smoke.py
```

## Run the server

<Tabs>
<TabItem value="stdio" label="Local stdio">

Use stdio for local desktop and CLI MCP clients that spawn the server as a subprocess:

```bash
AGENTA_API_KEY="agenta-api-key" \
AGENTA_API_URL="https://cloud.agenta.ai/api" \
agenta-mcp
```

Example client configuration:

```json
{
  "mcpServers": {
    "agenta": {
      "command": "agenta-mcp",
      "env": {
        "AGENTA_API_KEY": "agenta-api-key",
        "AGENTA_API_URL": "https://cloud.agenta.ai/api",
        "AGENTA_AUTH_SCHEME": "ApiKey"
      }
    }
  }
}
```

</TabItem>
<TabItem value="http" label="Internal HTTP">

Use Streamable HTTP when another internal service connects to the MCP server over the network:

```bash
AGENTA_API_KEY="agenta-api-key" \
AGENTA_API_URL="http://api:8000" \
AGENTA_MCP_TRANSPORT="streamable-http" \
MCP_HOST="0.0.0.0" \
MCP_PORT="8001" \
agenta-mcp
```

The server listens on:

```text
http://<host>:8001/mcp
```

Behind the provided Docker Compose Traefik snippets, clients reach the server at `/mcp/` and Traefik strips that prefix before forwarding to the MCP service.

</TabItem>
<TabItem value="docker" label="Docker">

Build the image locally:

```bash
docker build -t agenta-mcp .
```

Run it against Agenta Cloud or EU Cloud:

```bash
docker run --rm -p 8001:8001 \
  -e AGENTA_API_KEY="agenta-api-key" \
  -e AGENTA_API_URL="https://eu.cloud.agenta.ai/api" \
  -e AGENTA_AUTH_SCHEME="ApiKey" \
  -e AGENTA_MCP_TRANSPORT="streamable-http" \
  agenta-mcp
```

For persistent self-hosted deployment, paste the matching snippet into the Agenta Compose file:

- OSS: `clients/mcp-python/deploy/compose.snippet.yml`
- EE: `clients/mcp-python/deploy/compose.ee.snippet.yml`

Set `AGENTA_MCP_API_KEY` in the same environment file used by the stack.

</TabItem>

## Available tools

| Area | Tools |
| --- | --- |
| Applications | `list_applications`, `get_application`, `get_application_schema`, `list_application_templates`, `list_application_presets`, `create_application`, `update_application_prompt` |
| Evaluators | `list_evaluator_templates`, `list_evaluator_presets`, `list_evaluators`, `get_evaluator`, `create_evaluator` |
| Testsets | `list_testsets`, `get_testset`, `create_testset`, `upload_testset_file` |
| Evaluations | `list_evaluations`, `get_evaluation`, `create_evaluation` |
| Environments | `list_environments` |

After connecting a client, start with `list_applications` or `list_application_templates` to confirm the API key can read the intended project. `create_application` accepts `app_type="chat"`, `app_type="completion"`, or `app_type="custom"`; when omitted it falls back to `completion`. Custom apps require an explicit `uri`.

For evaluators, agents should first choose the evaluator family. Use `evaluator_type="auto"` with `auto_evaluator_type` values such as `exact_match`, `contains_json`, or `llm_as_a_judge`. Use `evaluator_type="human"` for feedback-style human evaluation; it defaults to the `feedback` template and `quality-rating` preset unless another preset is supplied.

## Security posture

The v1 server is internal-first. A hosted MCP endpoint with one `AGENTA_API_KEY` means every MCP caller acts as that key owner. Keep HTTP deployments private unless you implement separate client-to-MCP authentication and per-user or per-tenant credentials from MCP to Agenta.
