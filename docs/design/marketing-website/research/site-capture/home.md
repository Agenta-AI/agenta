### Section: THE PROBLEM — Why Most AI Teams Struggle
Five pain points listed:
1. Prompts scattered across Slack, Google Sheets, and emails
2. Product managers, developers, and domain experts work in silos
3. Vibe testing / yolo'ing changes to production
4. Zero visibility into whether experiments improve performance
5. Debugging feels like guesswork when things go wrong

### Section: THE SOLUTION — Your single source of truth for the whole team
Four capabilities:
- Centralize (prompts, evaluations, traces in one platform)
- Collaborate
- Create evaluations
- Monitor production systems

### Section: Experiment — Iterate your prompts with the whole team
Four feature cards:
1. Unified playground — compare prompts and models side-by-side
2. Complete version history — version prompts and track changes
3. Model agnostic — use any provider without vendor lock-in
4. Save production errors to test sets

### Section: Evaluate — Replace your guesswork with evidence
Four feature cards:
1. Automated evaluation
2. Integrate any evaluator (LLM-as-a-judge, built-in, code)
3. Evaluate full trace (each intermediate step)
4. Human evaluation

### Section: Observe — Debug your AI systems and gather user feedback
Four feature cards:
1. Trace every request
2. Annotate traces
3. Turn any trace into a test (single click)
4. Monitor performance and detect regressions

### Section: Collaborate — Bring PMs, experts, and devs into one workflow
Three feature cards:
1. A UI for your experts
2. Evals for everyone
3. Full API and UI parity
