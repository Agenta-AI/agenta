# Product OS Process

## Main loop

- frame the problem
- understand users and evidence
- define interfaces, use cases, flows, and requirements
- define acceptance, scenarios, and indicators
- review
- validate against user and product intent
- feed residue back into product knowledge

## Core activities

- framing
- discovery and research
- interface definition
- requirement definition
- acceptance definition
- traceability review
- product validation

## Actors and critics

- design actor = product designer
- implementation actor = product definer
- verification critic = product reviewer
- validation critic = product tester

## Likely skills or agents

- product framer
- user researcher
- competitor researcher
- interface shaper
- product reviewer
- scenario and acceptance author
- product analytics interpreter

## Main outputs

- refined product-facing interfaces
- use cases
- user flows
- product requirements
- acceptance criteria
- test scenarios
- metrics and indicators
