# Product OS State

## Main knowledge and state

- user understanding
- personas or target users
- jobs to be done
- pains and gains
- product-facing interfaces
- product definition
- requirements
- acceptance logic
- product analytics
- external docs as external knowledge interface

## Inputs

- existing product knowledge
- project knowledge
- agent knowledge
- process knowledge where relevant
- GTM feedback
- engineering constraints
- SRE signals on reliability or performance

## Outputs

- updated product state for the current work
- candidate product truth
- documentation updates where product-facing interfaces change
- feedback to GTM, Engineering, and SRE

## Important distinction

Product state is not only the current definition.

It also includes what users actually need, how the product is consumed, and how success is observed.
