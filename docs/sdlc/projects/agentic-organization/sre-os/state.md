# SRE OS State

## Main knowledge and state

- infrastructure topology
- environments
- deployment state
- observability state
- alerts and dashboards
- incidents and postmortems
- runbooks
- reliability and performance indicators
- operational security controls

## Inputs

- engineering knowledge
- product and GTM constraints when runtime behavior is user-visible
- project knowledge
- agent knowledge
- existing SRE knowledge

## Outputs

- updated runtime and operations state
- candidate SRE truth
- incident and reliability residue back to Engineering OS
- user-visible reliability residue back to Product OS
- externally relevant status residue back to GTM OS where needed

## Important distinction

SRE state is not just infrastructure inventory.

It also includes the operational knowledge needed to run, observe, recover, and improve the system over time.
