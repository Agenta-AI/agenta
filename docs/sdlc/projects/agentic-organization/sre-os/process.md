# SRE OS Process

## Main loop

- observe runtime state
- detect risk or failure
- respond
- recover
- harden
- improve automation and observability
- feed residue back into SRE and system knowledge

## Core activities

- deployment and rollout
- monitoring and alerting
- incident response
- post-incident learning
- capacity management
- production performance review
- reliability review
- runtime security operations

## Actors and critics

- design actor = SRE designer
- implementation actor = SRE operator
- verification critic = reliability reviewer
- validation critic = operational tester

## Likely skills or agents

- incident investigator
- observability improver
- deployment reviewer
- reliability reviewer
- runtime security reviewer
- capacity analyst
- postmortem synthesizer

## Main outputs

- runbook updates
- alerting and dashboard updates
- rollout and recovery guidance
- incident findings
- reliability improvements
- operational documentation updates
