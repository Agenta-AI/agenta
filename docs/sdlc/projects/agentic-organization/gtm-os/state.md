# Go-to-Market OS State

## Main knowledge and state

- market understanding
- segments and audiences
- positioning
- messaging
- pricing
- packaging
- launch state
- content inventory
- sales-enablement state
- funnel and growth telemetry

## Inputs

- existing GTM knowledge
- product knowledge
- project knowledge
- agent knowledge
- market signals
- revenue or funnel signals
- sales and DevRel feedback

## Outputs

- updated GTM state for the current work
- candidate GTM truth
- inputs back to Product OS on demand, friction, and messaging gaps
- inputs back to System OS on capability, packaging, or interface needs

## Important distinction

GTM state is not just communication artifacts.

It also includes the commercial and growth model around how the product is explained, adopted, and sold.
