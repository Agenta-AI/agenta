# Engineering OS Layer

## Purpose

Engineering OS defines how the application and platform are designed, built, reviewed, tested, and evolved.

## Main scope

- system interfaces
- subsystem boundaries
- ports and adapters
- domain and component design
- implementation
- testing
- code review
- engineering delivery process

## Main layers inside Engineering OS

- system layer
- subsystem layer
- component layer
- implementation layer

## Overlaps

### With Product OS

- product-facing interfaces become system interfaces
- requirements constrain contracts and behaviors
- acceptance drives validation

### With GTM OS

- capability truth for messaging and launch
- pricing or packaging enforcement implications
- docs and SDK realities

### With SRE OS

- deployability
- observability
- runtime behavior
- performance and reliability in production
- platform and infrastructure constraints

## Distinction

Engineering OS is the primary home for changing the software system itself.

It is not the primary home for running production operations.
