# Kubernetes OSS/EE self-hosting

This workspace tracks the work needed to make Agenta's Kubernetes self-hosting path support both OSS and EE cleanly.

## Goal

Provide a simple, non-hacky Kubernetes setup path for:

- OSS users
- self-hosted EE users

while preserving the existing Docker Compose deployment flow.

## Files

- `context.md` — problem statement, goals, non-goals, and constraints
- `plan.md` — agreed multi-PR execution plan
- `research.md` — current chart/runtime findings and EE-vs-OSS differences
- `status.md` — current status, branch/workstream tracking, blockers, and next steps
