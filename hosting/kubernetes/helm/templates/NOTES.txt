Agenta is accessible via Ingress:

  Determine the ingress IP:

    API:      http://<INGRESS_IP>/api/
    Services: http://<INGRESS_IP>/services/

{{- else }}

Agenta services are available via ClusterIP. Use port-forward to access them:

  # Web UI
  kubectl port-forward svc/{{ include "agenta.fullname" . }}-web {{ include "agenta.web.port" . }}:{{ include "agenta.web.port" . }} -n {{ .Release.Namespace }}

  # API
  kubectl port-forward svc/{{ include "agenta.fullname" . }}-api {{ include "agenta.api.port" . }}:{{ include "agenta.api.port" . }} -n {{ .Release.Namespace }}
{{- end }}

To check the status of all pods:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

{{- if eq (include "agenta.alembic.enabled" .) "true" }}

Database migrations run as a post-install/post-upgrade hook. Check the migration job:
  kubectl get jobs -n {{ .Release.Namespace }} -l app.kubernetes.io/component=alembic
  kubectl logs job/{{ include "agenta.fullname" . }}-alembic -n {{ .Release.Namespace }}
{{- end }}

{{- if eq (include "agenta.postgresql.enabled" .) "true" }}

Bundled PostgreSQL is enabled. Bitnami is configured to read the password from:
  global.postgresql.auth.existingSecret: {{ include "agenta.pgauthSecretName" . }}
{{- if ne (include "agenta.pgauthSecretName" .) $secrets.existingSecret }}

WARNING: global.postgresql.auth.existingSecret resolves to "{{ include "agenta.pgauthSecretName" . }}"
but secrets.existingSecret is "{{ $secrets.existingSecret }}".
The Bitnami PostgreSQL subchart may fail to find the password.
Set: --set global.postgresql.auth.existingSecret={{ $secrets.existingSecret }}
{{- end }}
{{- end }}
{{- else }}

IMPORTANT: Three secrets are REQUIRED. Set them in your values file:

  agenta:
    authKey: "<your-auth-key>"
    cryptKey: "<your-crypt-key>"
  postgres:
    password: "<your-postgres-password>"

Then install with:

  helm install agenta hosting/kubernetes/helm \
    -f hosting/kubernetes/oss/values.oss.example.yaml
