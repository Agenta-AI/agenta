Thank you for installing {{ .Chart.Name }}!

Your release is named: {{ .Release.Name }}

{{- if .Values.ingress.enabled }}

Agenta is accessible via Ingress:
{{- if .Values.ingress.host }}
  Web UI:   http{{ if .Values.ingress.tls }}s{{ end }}://{{ .Values.ingress.host }}/
  API:      http{{ if .Values.ingress.tls }}s{{ end }}://{{ .Values.ingress.host }}/api/
  Services: http{{ if .Values.ingress.tls }}s{{ end }}://{{ .Values.ingress.host }}/services/
{{- else }}
  Determine the ingress IP:
    kubectl get ingress {{ include "agenta.fullname" . }} -n {{ .Release.Namespace }}

  Then access:
    Web UI:   http://<INGRESS_IP>/
    API:      http://<INGRESS_IP>/api/
    Services: http://<INGRESS_IP>/services/
{{- end }}
{{- else }}

Agenta services are available via ClusterIP. Use port-forward to access them:

  # Web UI
  kubectl port-forward svc/{{ include "agenta.fullname" . }}-web {{ .Values.web.port }}:{{ .Values.web.port }} -n {{ .Release.Namespace }}

  # API
  kubectl port-forward svc/{{ include "agenta.fullname" . }}-api {{ .Values.api.port }}:{{ .Values.api.port }} -n {{ .Release.Namespace }}
{{- end }}

To check the status of all pods:
  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

{{- if .Values.alembic.enabled }}

Database migrations run as a post-install/post-upgrade hook. Check the migration job:
  kubectl get jobs -n {{ .Release.Namespace }} -l app.kubernetes.io/component=alembic
  kubectl logs job/{{ include "agenta.fullname" . }}-alembic -n {{ .Release.Namespace }}
{{- end }}

{{- if .Values.secrets.existingSecret }}

You are using a pre-created Secret: {{ .Values.secrets.existingSecret }}
Ensure it contains the following keys:
  - AGENTA_AUTH_KEY
  - AGENTA_CRYPT_KEY
  - POSTGRES_PASSWORD
  - SUPERTOKENS_API_KEY  (optional)
{{- if .Values.postgresql.enabled }}

Bundled PostgreSQL is enabled. Bitnami is configured to read the password from:
  global.postgresql.auth.existingSecret: {{ include "agenta.pgauthSecretName" . }}
{{- if ne (include "agenta.pgauthSecretName" .) .Values.secrets.existingSecret }}

WARNING: global.postgresql.auth.existingSecret resolves to "{{ include "agenta.pgauthSecretName" . }}"
but secrets.existingSecret is "{{ .Values.secrets.existingSecret }}".
The Bitnami PostgreSQL subchart may fail to find the password.
Set: --set global.postgresql.auth.existingSecret={{ .Values.secrets.existingSecret }}
{{- end }}
{{- end }}
{{- else }}

IMPORTANT: Three secrets are REQUIRED. Install with:

  helm install agenta hosting/helm/agenta-oss \
    --set secrets.agentaAuthKey=$(openssl rand -hex 32) \
    --set secrets.agentaCryptKey=$(openssl rand -hex 32) \
    --set secrets.postgresPassword=$(openssl rand -hex 16)
{{- end }}
